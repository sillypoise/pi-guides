Prefer direct, concrete language.

Distinguish observed facts from inferences, recommendations, and unresolved questions.

Do not overstate certainty. When confidence is limited, say so briefly and explain what would verify it.

State key assumptions when they materially affect the diagnosis, plan, or recommendation.

Prefer concise answers by default. Expand when task risk, implementation complexity, or user request justifies more detail.

When recommending an approach, make the main tradeoffs, risks, and unknowns explicit.

Avoid vague filler, inflated language, and unsupported claims.
