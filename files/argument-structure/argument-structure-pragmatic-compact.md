# Argument-Structure Playbook - Pragmatic / Compact

Compact derivative of `argument-structure/argument-structure-pragmatic-full.md`.
Section tags retain the same meaning.

## Scope

- Decision, proposal, and problem-analysis documents.
- Not for all docs by default; use only when explicit decision logic is needed.

## ARG-SCQA

- Build intros in order: Situation -> Complication -> Question -> Answer.
- Keep one primary complication and one explicit question.
- Place top-line answer immediately after question.

Failure checks:

- Intro lists topics but never asks a real decision question.
- Recommendation appears late and forces reader inference.

## ARG-PYR

- Start with one key line claim.
- Group supports as same-kind sibling points.
- Ensure each child level directly answers the parent-level question.

Failure checks:

- Mixed sibling kinds (reasons + actions + observations).
- Supporting points do not defend the parent claim.

## ARG-ORDER

- Use one order type per sibling set:
  - deductive
  - chronological
  - structural
  - priority

Failure checks:

- Order type shifts mid-group without reason.
- Critical points are buried under low-impact details.

## ARG-ISSUE

- Separate analysis from recommendation drafting.
- Use stable criteria across options.
- Make recommendation traceable to branch evidence.

Failure checks:

- Recommendation cannot be traced to analysis branches.
- Criteria change mid-analysis to favor a preferred outcome.

## Review Checklist

- Intro contains explicit Question and Answer.
- Recommendation is supported by same-kind grouped reasons.
- Order type is consistent within each group.
- Tradeoffs and assumptions are explicit.
- Next actions include owner and decision implications.
